# AssetFlow

**Enterprise Asset & Resource Management System**

## Problem Statement
We chose to build **AssetFlow** to solve the complex challenges organizations face when tracking physical assets, managing shared resources, and enforcing accountability. Many companies struggle with spreadsheets and siloed systems; AssetFlow provides a unified, role-based platform that handles the entire lifecycle—from procurement and allocation to maintenance, auditing, and retirement.

## Tech Stack
- **Frontend:** React (Vite) + TailwindCSS v4
- **Backend:** Node.js + Express
- **Database:** PostgreSQL
- **ORM:** Prisma
- **Auth:** JWT + bcrypt (with Role-Based Access Control middleware)

## Setup Instructions

1. **Clone the repository:**
   ```bash
   git clone <YOUR_REPO_URL>
   cd assetflow
   ```

2. **Install dependencies:**
   Open two terminal windows.
   - Terminal 1 (Frontend):
     ```bash
     cd frontend
     npm install
     ```
   - Terminal 2 (Backend):
     ```bash
     cd server
     npm install
     ```

3. **Environment Setup:**
   - In the `frontend` folder, copy `.env.example` to `.env` (contains `VITE_API_BASE_URL=http://localhost:5000/api`).
   - In the `server` folder, create a `.env` file and add your local PostgreSQL connection string:
     ```env
     DATABASE_URL="postgresql://postgres:postgres@localhost:5432/assetflow?schema=public"
     JWT_SECRET="your_secret_key"
     PORT=5000
     ```

4. **Database Setup & Seeding:**
   Ensure your PostgreSQL server is running. Then, in the `server` folder:
   ```bash
   npx prisma migrate dev
   npx prisma db seed
   ```
   *(Note: The seed script is critical as it bootstraps the initial System Admin account required to promote other users!)*

5. **Run the Application:**
   Start both dev servers.
   - Frontend: `npm run dev`
   - Backend: `npm run dev`

## Screens Overview
1. **Login:** Secure email/password authentication.
2. **Signup:** Employee self-registration.
3. **Dashboard:** High-level KPIs and urgent action items.
4. **Org Setup:** Directory to manage departments and promote user roles (Admin only).
5. **Asset Registry:** Full inventory of all hardware, furniture, and vehicles.
6. **Allocations:** Track which employee currently holds which asset.
7. **Bookings:** Calendar-based reservation system for shared resources (e.g., Meeting Rooms, Vehicles).
8. **Maintenance:** Kanban board to track and resolve hardware issues.
9. **Audit Cycles:** Periodic inventory checks and reconciliation.
10. **Reports & Logs:** Utilization analytics and historical activity logs.
